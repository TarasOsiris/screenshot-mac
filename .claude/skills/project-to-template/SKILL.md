# Convert Project to Template

Turn an existing user project into a template in `screenshot/Templates.bundle/`.

Usage: `/project-to-template <project-name> [template-folder-name]`

If `$ARGUMENTS` is empty, ask the user for the project name.

## Step 1: Locate the project

Projects are stored in:
```
~/Library/Mobile Documents/iCloud~xyz~tleskiv~screenshot/Documents/screenshot/
```

1. Read `projects.json` from that directory to find the project by name (case-insensitive match).
2. The project's data is at `projects/<project-id>/` (containing `project.json` and optionally `resources/`).
3. If multiple projects match or none found, ask the user to clarify.

## Step 2: Determine template folder name

If a second argument was provided, use it as the folder name. Otherwise, derive it from the project name:
- Lowercase
- Replace spaces with hyphens
- Remove special characters

Check that the folder doesn't already exist in `screenshot/Templates.bundle/`. If it does, ask the user whether to overwrite.

## Step 3: Copy project to Templates.bundle

```bash
cp -R "<project-path>" screenshot/Templates.bundle/<folder-name>
```

Ensure the `resources/` subdirectory exists (create it if missing).

## Step 4: Generate preview image

The preview is generated by rendering actual row canvases via `ExportService`. Use the Python fallback generator:

```bash
python3 -c "
import sys
sys.path.insert(0, '<repo-root>/tools')
from gen_template import generate_preview
generate_preview(
    '<repo-root>/screenshot/Templates.bundle/<folder-name>/project.json',
    '<repo-root>/screenshot/Templates.bundle/<folder-name>/preview.png'
)
"
```

Replace `<repo-root>` with the actual absolute path to the repository root.

For high-quality previews rendered from actual SwiftUI views, use the Debug menu > "Regenerate All Previews" in the running app (DEBUG builds only). This uses `DebugTemplateService.regenerateAllPreviews()` which renders full row canvases via `ExportService.renderRowImage` and composites all rows into the preview.

## Step 5: Verify

1. Confirm the template directory contains `project.json`, `preview.png`, and `resources/`.
2. Build the project to verify:
   ```bash
   xcodebuild -scheme screenshot -destination 'platform=macOS' build
   ```

## Step 6: Report

Print a summary:
- Source project name and ID
- Template folder name
- Number of rows and templates per row (parse from project.json)
